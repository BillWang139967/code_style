## Code review 步骤

> * Step 1：系统全貌
>   * 模块划分的逻辑，模块之间的关系
> * Step 2：模块级别
>   * 看清模块内的逻辑
>   * 关键数据，关键的类、函数
> * Step 3：类、函数内部的逻辑
> * Step 4：细节
>   * Layout，命名，...

## 软件工程
> * 一流代码的特性
>   * 高效（Fast）
>   * 鲁棒（Solid and Robust）
>   * 简洁（Maintainable and Simple）
>   * 共享（Re-usable）
>   * 可测试（Testable）
>   * 可移植（Portable）
>   * 可监控（Monitorable）
>   * 可运维（Operational）
>     * 部署 Easy to deploy
>     * 变更 Easy to changes
>     * 维护 Easy to maintain
>     * 监控 Easy to observable
>   * 可扩展（Scalable & Extensible）
> * 需求分析 & 系统设计
>   * 需求分析：定义系统 / 软件黑盒的行为（external,what）
>   * 系统设计：设计系统 / 软件白盒的机制（internal，how&why）
